# 📘 คู่มือติดตั้ง JOJO+ แบบง่ายที่สุด (สำหรับคนไม่เคยทำ)

> ระบบนี้ทำงานบน **Google Sheet + Apps Script** ฟรี ไม่ต้องลงโปรแกรมอะไรในเครื่อง
> ใช้แค่ **บัญชี Google (Gmail)** กับ **เบราว์เซอร์ (Chrome)** เท่านั้น
> ทำตามทีละขั้น ใช้เวลาประมาณ **15–20 นาที**

---

## 🧾 เตรียมของก่อนเริ่ม

1. บัญชี Gmail 1 บัญชี (บัญชีนี้จะกลายเป็น "แอดมิน" ของระบบ)
2. ไฟล์โค้ด 4 ไฟล์ (อยู่ในโฟลเดอร์ที่ได้รับมา):
   - `Code.gs`
   - `Backup.gs`
   - `Index.html`
   - `appsscript.json`
3. เปิด Chrome แล้ว **ล็อกอิน Gmail บัญชีที่จะใช้ให้เรียบร้อย**

> 💡 ถ้าเปิดหลายบัญชี Google พร้อมกันในเบราว์เซอร์ มักทำให้งง — แนะนำเปิด **บัญชีเดียว** ตอนติดตั้ง

---

## ① สร้าง Google Sheet เปล่า (ฐานข้อมูลของระบบ)

1. เข้า https://sheets.google.com → กด **＋ ว่าง (Blank)**
2. ตั้งชื่อไฟล์ว่า `JOJO+ Phishing Simulation` (ตั้งชื่ออื่นก็ได้)

**เท่านี้พอ ยังไม่ต้องพิมพ์อะไรในตาราง** — เดี๋ยวระบบสร้างหัวตารางให้เอง

---

## ② เปิดหน้าเขียนสคริปต์ (Apps Script)

1. ในหน้า Google Sheet เดิม ไปที่เมนูบน → **ส่วนขยาย (Extensions)** → **Apps Script**
2. จะเปิดแท็บใหม่ขึ้นมา (นี่คือที่วางโค้ด)

---

## ③ วางโค้ด 4 ไฟล์ให้ครบ (ชื่อต้องตรงเป๊ะ)

ในหน้า Apps Script ทางซ้ายจะเห็นรายชื่อไฟล์ ให้ทำแบบนี้:

| ต้องมีไฟล์ชื่อ | ชนิด | วิธีสร้าง |
|---|---|---|
| `Code` (`.gs`) | สคริปต์ | มีอยู่แล้ว → ลบของเดิมในนั้นทิ้ง แล้ววางเนื้อหาจาก `Code.gs` |
| `Backup` (`.gs`) | สคริปต์ | กด **＋** ข้างคำว่า Files → **Script** → ตั้งชื่อ `Backup` → วางเนื้อหาจาก `Backup.gs` |
| `Index` (`.html`) | HTML | กด **＋** → **HTML** → ตั้งชื่อ `Index` → วางเนื้อหาจาก `Index.html` |
| `appsscript` (`.json`) | มานิเฟสต์ | ดูวิธีเปิดด้านล่าง ⤵ แล้ววางเนื้อหาจาก `appsscript.json` |

**วิธีเปิดไฟล์ `appsscript.json`** (ปกติถูกซ่อนไว้):
- กดรูปเฟือง ⚙️ **การตั้งค่าโปรเจกต์ (Project Settings)** ทางซ้าย
- ติ๊กถูก ☑ **"แสดงไฟล์ manifest 'appsscript.json' ในเครื่องมือแก้ไข"**
- กลับไปหน้า Editor จะเห็นไฟล์ `appsscript.json` แล้ว → เปิดแล้ววางเนื้อหาทับ

> ⚠️ **สำคัญมาก:** ไฟล์ HTML ต้องชื่อ `Index` (ตัว I ใหญ่) เป๊ะ ไม่งั้นระบบจะขึ้น error "ไม่พบไฟล์ Index"

> 💡 วางโค้ดแต่ละไฟล์เสร็จ อย่าลืมกด **💾 บันทึก (Save)** ทุกไฟล์

---

## ④ ตั้งค่า CONFIG (แก้แค่ 1 จุด หรือไม่แก้เลยก็ได้)

เปิดไฟล์ `Code.gs` เลื่อนขึ้นบนสุด จะเห็นบล็อก `const CONFIG = { ... }`

**ถ้าคุณติดตั้งบนบัญชีตัวเอง และอยากให้ตัวเองเป็นแอดมิน → ไม่ต้องแก้อะไรเลย ข้ามไปข้อ ⑤ ได้**
(ระบบจะถือว่า "เจ้าของ Google Sheet" = แอดมินอัตโนมัติ)

ถ้าได้โค้ดต่อมาจากคนอื่นแล้วในนั้นมีอีเมล/URL ของคนเดิมอยู่ ให้เคลียร์เป็นค่าว่างแบบนี้:

```javascript
const CONFIG = {
  developerEmail: '',        // ปล่อยว่าง = เจ้าของชีตเป็นแอดมินเอง
  adminEmails: [],           // แอดมินเพิ่มเติม (ปล่อยว่างได้)
  spreadsheetId: '',         // ปล่อยว่างได้
  webAppUrl: '',             // ปล่อยว่างไปก่อน (ใส่ทีหลังข้อ ⑥ ถ้าจำเป็น)
  trainingWebAppUrl: '',     // ⚠ ห้ามใส่ URL ของคนอื่น! ปล่อยว่างไปก่อน
  seedDemo: false            // false = ฐานข้อมูลสะอาด ไม่มีข้อมูลตัวอย่าง
};
```

> ⚠️ ถ้าปล่อย `trainingWebAppUrl` เป็น URL ของคนอื่น **ผลการอบรมจะไหลไปลงชีตของคนนั้น** — ต้องเป็นค่าว่างหรือ URL ของตัวเองเท่านั้น

กด **💾 บันทึก**

## ⑤ รันปุ่มเดียวเพื่อสร้างฐานข้อมูล (setupDatabase)

1. ด้านบนของหน้า Apps Script มีช่องเลือกฟังก์ชัน (dropdown) → เลือก **`setupDatabase`**
2. กดปุ่ม **▶ เรียกใช้ (Run)**
3. **ครั้งแรกจะขึ้นให้ขออนุญาต** — ทำตามนี้:
   - กด **ตรวจสอบสิทธิ์ (Review permissions)**
   - เลือกบัญชี Gmail ของคุณ
   - ถ้าขึ้นจอ "Google ยังไม่ได้ยืนยันแอปนี้" → กด **ขั้นสูง (Advanced)** → **ไปที่ ... (unsafe)** → **อนุญาต (Allow)**
   - *(หน้าจอนี้เป็นเรื่องปกติของสคริปต์ที่เราเขียนเอง ไม่ใช่ไวรัส)*
4. รอจนขึ้นข้อความ **"ดำเนินการเสร็จสมบูรณ์"** ด้านล่าง

✅ ตอนนี้กลับไปดูใน Google Sheet จะเห็นแท็บต่างๆ ถูกสร้างให้อัตโนมัติ (Customers, EmailList, Queue, ฯลฯ)

> 💡 ปุ่มนี้กดซ้ำได้ไม่พัง — ถ้าไม่แน่ใจกดใหม่ได้เสมอ

## ⑥ เผยแพร่เป็นเว็บแอป (Deploy) — ต้องทำ 2 ตัว

ในหน้า Apps Script มุมขวาบน กด **เผยแพร่ (Deploy)** → **การทำให้ใช้งานได้ใหม่ (New deployment)**
กดรูปเฟือง ⚙️ ข้างคำว่า "เลือกประเภท" → เลือก **เว็บแอป (Web app)**

### 🔵 ตัวที่ 1 — ADMIN (หน้าจัดการของแอดมิน)

ตั้งค่าดังนี้ แล้วกด **เผยแพร่ (Deploy)**:

| ช่อง | เลือก |
|---|---|
| คำอธิบาย (Description) | `Admin` (พิมพ์อะไรก็ได้) |
| ดำเนินการในนาม (Execute as) | **ผู้ใช้ที่เข้าถึงเว็บแอป (User accessing)** |
| ผู้มีสิทธิ์เข้าถึง (Who has access) | **ทุกคนที่มีบัญชี Google (Anyone with Google account)** |

→ ก๊อป **URL ที่ลงท้ายด้วย `/exec`** เก็บไว้ นี่คือ **ลิงก์หน้าแอดมิน**

### 🟢 ตัวที่ 2 — TRAINING (หน้าอบรมสำหรับพนักงาน)

กด **เผยแพร่ (Deploy)** → **New deployment** อีกครั้ง → เว็บแอป → ตั้งค่า:

| ช่อง | เลือก |
|---|---|
| คำอธิบาย | `Training` |
| ดำเนินการในนาม (Execute as) | **ฉัน / เจ้าของ (Me)** |
| ผู้มีสิทธิ์เข้าถึง | **ทุกคน (Anyone)** ← ไม่ต้องล็อกอิน |

→ ก๊อป **URL `/exec`** เก็บไว้ นี่คือ **ลิงก์หน้าอบรม** (ลิงก์ที่จะใส่ในอีเมลให้พนักงาน)

> ⚠️ จำให้แม่น: ตัว Training ต้องเป็น **Execute as = ฉัน (Me)** และ **Anyone** เท่านั้น
> เพื่อให้พนักงานเปิดหน้าอบรมได้โดยไม่ต้องล็อกอิน Google

---

## ⑦ ใส่ URL กลับเข้า CONFIG (ทำเมื่อลิงก์ในเมลเพี้ยนเท่านั้น)

ปกติปล่อยว่างระบบหาเองได้ **แต่ถ้าลิงก์ในเมลกดแล้วเพี้ยน** ให้เอา URL 2 ตัวจากข้อ ⑥ (ของตัวเองเท่านั้น) มาใส่:

```javascript
  webAppUrl: 'วาง URL /exec ของตัว ADMIN',
  trainingWebAppUrl: 'วาง URL /exec ของตัว TRAINING',
```

กด **💾 บันทึก** → แล้ว **เผยแพร่ตัว ADMIN ใหม่** (Deploy → Manage deployments → ✏️ แก้ไข → New version → Deploy)

---

## ✅ เสร็จแล้ว! ลองใช้งาน

1. เปิด **ลิงก์หน้าแอดมิน** (URL ตัว Admin) → ล็อกอินด้วย Gmail ที่ติดตั้ง
   → จะเห็นเมนูครบ (Email List, Campaign, Schedule, Report ฯลฯ) = คุณเป็นแอดมินแล้ว
2. ไปเมนู **Email List** → ใส่รายชื่ออีเมลเป้าหมาย
3. ไปเมนู **Campaign** → สร้างคิว → กดส่ง
4. เปิด **ลิงก์หน้าอบรม** เพื่อทดสอบว่าหน้าอบรมเปิดได้โดยไม่ต้องล็อกอิน
5. ดูผลที่เมนู **Report**

## 🆘 เจอปัญหาบ่อยๆ

| อาการ | วิธีแก้ |
|---|---|
| ขึ้น error **"ไม่พบไฟล์ Index"** | ไฟล์ HTML ตั้งชื่อผิด ต้องเป็น `Index` (I ใหญ่) เป๊ะ |
| หน้าอบรมกดลิงก์แล้ว **เด้งให้ล็อกอิน** | ตัว Training ตั้งเป็น "ต้องล็อกอิน" → แก้เป็น **Execute as: Me + Anyone** แล้ว New version |
| **ลิงก์ในเมลเพี้ยน / เปิดไม่ติด** | ทำข้อ ⑦ (ใส่ URL กลับเข้า CONFIG) |
| **ผลอบรมไปลงชีตคนอื่น** | `trainingWebAppUrl` ยังเป็น URL ของคนเดิม → เคลียร์เป็น `''` หรือใส่ของตัวเอง |
| แก้โค้ดแล้ว **หน้าเว็บไม่เปลี่ยน** | ต้อง **เผยแพร่ New version** ก่อน (แค่ Save เฉยๆ เว็บยังเป็นเวอร์ชันเก่า) |
